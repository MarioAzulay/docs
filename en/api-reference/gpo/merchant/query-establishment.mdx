## Query Establishment

This service allows you to query information about an establishment. An establishment is always associated with a merchant.

### Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token in the format: `Bearer {token}`
</ParamField>

### Path Parameters

<ParamField path="merchantId" type="string" required>
  Merchant ID
</ParamField>

<ParamField path="establishmentId" type="string" required>
  Establishment ID
</ParamField>

### Response

  <Expandable title="Properties">
    <ResponseField name="id" type="string">
      Establishment ID
    </ResponseField>
    <ResponseField name="name" type="string">
      Establishment name
    </ResponseField>
    <ResponseField name="createdDate" type="string">
      Creation date
    </ResponseField>
    <ResponseField name="updatedDate" type="string">
      Update date
    </ResponseField>
    <ResponseField name="contact" type="object">
      <Expandable title="Properties">
        <ResponseField name="email" type="string">
          Contact email
        </ResponseField>
        <ResponseField name="phone" type="string">
          Contact phone
        </ResponseField>
        <ResponseField name="address" type="string">
          Contact address
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="merchant" type="object">
      <Expandable title="Properties">
        <ResponseField name="id" type="string">
          Merchant ID
        </ResponseField>
        <ResponseField name="name" type="string">
          Merchant name
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="pointsOfSale" type="array">
      <Expandable title="Items">
        <ResponseField name="id" type="string">
          Terminal ID
        </ResponseField>

          Terminal name
        </ResponseField>
        <ResponseField name="state" type="string">
          Terminal state
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl --request GET \
  --url "https://gpo-sandbox.izipay.ao/v1/merchants/185737/establishments/12345" \
  --header "Authorization: Bearer YOUR_TOKEN"
```

### Response Codes

| Status Code | Description |
|-------------|-------------|
| 200 | OK - Establishment information retrieved successfully |
| 401 | Unauthorized - Invalid token |
| 404 | Not Found - Merchant or establishment not found |
